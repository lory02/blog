你不知道的js（上）- 作用域和闭包

发表于 2019-11-22 | 分类于 books

花了几天时间断断续续的读完了这本书，这确实是一本进阶和巩固js的书。

这本书的一开始的节奏，其实有点让我犯困，不过第二部分实在是惊艳（从动态和静态来解析this，从[[prototype]]属性来关联对象，介绍委托设计模式…），它的观点很直接，从语言本身的设计角度来剖析对象和原型，用一些更深层的原理来解析，Javascript中的代码细节。发现之前，自己遇到一些的困惑，在这里都得到了解答。

不过不建议一开始初学就看这本书，建议在对语言有了一定了解之后，再来读这本书，肯定会有收获。

第一章作用域是什么？

编译原理
  • 分词/词法分析
    • 将代码分解成词法单元（分割代码—面包原料）
      • 分词和词法分析的区别：有状态还是无状态
  • 解析/语法分析
    • 将词法单元流转换成一个AST抽象语法树
      • {type : ‘’,value :’’ …} ， 识别每个单元具体作用（盐-调料，烤箱-厨具）
      • AST（img）
  • 代码生成
    • 生成机器码

总结： 分解成词法单元=》 构建语法树 =》 生成机器码

作用域

学习作用域的方式，是要取了解在代码编译和执行的过程中，作用域扮演何种角色。

  • 编译 var a （是否需要创建变量a）
    • 在当前作用域中寻找变量a，如果无则声明，有则忽略
  • 执行 a = 1 （给哪个变量a赋值）
    • 在当前作用域中寻找变量a，有则赋值，没有则沿着作用域链向上查找
    • 如果在全局作用域中也未找到a，则新建一个全局变量a

总结: 变量的赋值操作会执行两个动作，首先编译器会在当前作用域中声明一个变量（如 果之前没有声明过），然后在运行时引擎会在作用域中查找该变量，如果能够找到就会对 它赋值。

LHS 和 RHS

LHS : 左侧查询，用于赋值 var a = 1

RHS : 右侧查询，用于取值 console.log(a)

异常

第二章词法作用域

词法阶段

编译器的第一个工作阶段，叫作词法化（也叫单词化-词法分析）。词法化的过程会对源代码中的字符进行检查，如果是有状态的解析过程，还会赋予单词语义。

词法作用域

词法作用域就是定义在词法阶段的作用域。（也就是在执行之前就确定了作用域）换句话说，词法作用域是由你在写 代码时将变量和块作用域写在哪里来决定的，因此当词法分析器处理代码时会保持作用域 不变（大部分情况下是这样的。

作用域查找

当前的作用域位置，向外查找至全局作用域

函数作用域

函数中没有var是全局变量，有var是局部变量

欺骗词法

  • eval( )

    • 执行时才运行eval中的代码，可能会改变原有作用域结构

    • 在严格模式的程序中，eval(..) 在运行时有其自己的词法作用域，意味着其 中的声明无法修改所在的作用域

  • with()

    • with 通常被当作重复引用同一个对象中的多个属性的快捷方式，可以不需要重复引用对象 本身

    • with 可以将一个没有或有多个属性的对象处理为一个完全隔离的词法作用域 (相当于区别于函数或者全局作用域的第三种作用域，即对象作用域

总结

词法作用域意味着作用域是由书写代码时函数声明的位置来决定的。编译的词法分析阶段基本能够知道全部标识符在哪里以及是如何声明的，从而能够预测在执行过程中如何对它 们进行查找。

第三章 函数作用域和块作用域

函数作用域

  • 隐藏内部实现： 私有化变量，防止外部修改私有变量

  • 规避冲突：防止同名标识符之间的冲突

  • 函数表达式和函数声明的差别

    • 表达式：包装函数的声明以 (function… 而不仅是以 function… 开始

      • foo 被绑定在所在作用域中，可以直接通过 foo() 来调用它

    • 声明：function 开头

      • foo 被绑定在函数表达式自身的函数中而不是所在作用域中

  • IIFE，立即执行函数表达式

块作用域
  • with
    • 用 with 从对象中创建出的作用域仅在 with 声明中而非外 部作用域中有效
  • try/catch
    • try/catch 的 catch 分句会创建一个块作 用域，其中声明的变量仅在 catch 内部有效
    • ⭐ 在同一个作用域使用多个catch，如果都使用err作为形参，很多静态工具会发出警告，但其实所有变量都被安全地限制在块级作用域内，很多开发者会将 catch 的参数命名为 err1、 err2 等。也有开发者干脆关闭了静态检查工具对重复变量名的检查。
  • let
    • let 关键字可以将变量绑定到所在的任意作用域中（通常是 { .. } 内部）
    • for 循环
  • const
    • const，同样可以用来创建块作用域变量，但其值是固定的 （常量）。之后任何试图修改值的操作都会引起错误

第四章 提升

当你看到 var a = 2; 时，可能会认为这是一个声明。但 JavaScript 实际上会将其看成两个 声明：var a; 和 a = 2;。第一个定义声明是在编译阶段进行的。第二个赋值声明会被留在 原地等待执行阶段。

函数优先 ： 函数声明和变量都会被提升，函数会首先被提升，然后才是变量（这个细节可以出现在有多个 “重复”声明的代码中）

第五章 作用域闭包

闭包

闭包使得函数可以继续访问定义时的词法作用域。(作用域查找和闭包的区别)

模块

将功能封装在函数中，通过函数调用的返回值暴露接口，通过返回值来获取模块的中方法，或者取值。（利用了闭包的特性）

  • 一个具有函数属性的对象本身并不是真正的模块
  • 一个函数调用所返回的，只有数据属性而没有闭包函数的对象并不是真正的闭包
⭐现代的模块机制

大多数模块依赖加载器 / 管理器本质上都是将这种模块定义封装进一个友好的 API

  • 核心： 函数调用（封装） + 返回对象（接口）
    • 模块函数（内部定义多个变量或函数），调用模块函数
    • 返回对象（API接口），对象内的属性是对内部变量或函数的一个引用

定义模块

未来的模块机制

es6模块 export 和 import

export : 将当前模块的一个标识符（变量、函数）导出为公 共 API

import: 将一个模块中的一个或多个 API 导入到当前作用域中，并分别绑定在一个变量 上

module: 会将整个模块的 API 导入并绑定到一个变量上

总结

闭包 ：

当函数可以记住并访问所在的词法作用域，即使函数是在当前词法作用域之外执行，这时 就产生了闭包。

模块的两个主要的特征：

  1. 为创建内部作用域二调用了一个包装函数
  2. 包装函数的返回值必须至少包括一个内部函数引用，这样就会创建涵盖整个包装函数内部作用域的闭包

动态作用域

动态作用域不关心函数和作用域在何处被声明，只关心它们从何处调用。动态作用域是基于调用栈的，而不是代码中的作用域嵌套。

词法作用域和动态作用域的差别

词法作用域是在写代码或者说定义时确定的，而动态作用域是在运行时确定 的。（this 也是！）词法作用域关注函数在何处声明，而动态作用域关注函数从何处调用
