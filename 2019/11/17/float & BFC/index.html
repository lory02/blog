float & BFC

发表于 2019-11-17 | 分类于 css

BFC

BFC：块格式化上下文（Block Formatting Context）

BFC会包含创建它的元素内部的所有内容。具有BFC特性的元素可以看做是隔离了的独立容器，容器里面的元素不会在布局上影响到外面的元素 (A block formatting context contains everything inside of the element creating it)

创建BFC的方法

  • 根元素

  • float的值不为none

  • overflow的值不为visible的块元素

    • 父元素：overflow：hidden | overflow: auto | overflow: visible

  • display: inline-block、table-cell、table-caption

  • display：flow-root

    • 一个新的 display 属性的值，它可以创建无副作用的BFC

  • position : absolute或fixed

BFC的应用
  1. 清除浮动
  2. 防止相邻兄弟元素margin合并

float

folat简介

float ，指定一个元素应沿其容器的左侧或右侧放置，允许文本和内联元素环绕它。该元素从网页的常规流中移除，尽管仍然保持部分的流动性。

当一个元素浮动之后，它会被移出常规流，然后向左或者向右平移，一直平移直到碰到了所处的容器的边框，或者碰到另外一个浮动的元素

  • 浮动元素触发BFC，所以浮动元素外边距不会合并

  • 除了块级元素，元素width都会尽然窄，元素设置float，如果元素没有指定width，会被压缩成0。为了浮动非替换元素，需要给定一个指定width。

  • 浮动元素的包含块，是最近的块级祖先元素

  • 与浮动元素重叠，行内元素浮于上方，块级元素在下方

  • 块级元素不会浮动到之前的块级元素，后面的块级元素会忽略浮动元素，会重叠在float元素底部。

  • 取值： left | right | none | inline-start | inline-end

如何清除浮动带来的影响？

清除浮动

clear：指定一个元素是否必须移动(清除浮动后)到在它之前的浮动元素下面。clear 属性适用于浮动和非浮动元素。

值：none | left | right | both | inline-start | inline-end

  • left：元素被向下移动用于清除之前的左浮动
  • right：元素被向下移动用于清除之前的右浮动

闭合浮动

子元素设置为float，父元素塌陷，height=0

解决方法

  • clearfix：通过伪元素在父元素中建立一个空的块级元素 ，让其 clear : both（注意是用在最近的块级祖先）

  • 父元素触发BFC

    overflow: hidden/ auto/ scroll;

    display: inline-block/ table-cell/ table/ flow-root;

    position: absolute/ fixed;

    float: left/ right;
