redux原则

  • store是唯一的

  • 只有store能够改变自己的内容，reducer只可以接收state，但是绝不能修改state

  • reducer必须是纯函数

    纯函数：一个函数的返回结果只依赖于它的参数，并且在执行过程里面没有副作用，我们就把这个函数叫做纯函数

    纯函数很严格，也就是说你几乎除了计算数据以外什么都不能干，计算的时候还不能依赖除了函数参数以外的数据。

    • js中的不可变类型和可变类型（obj）
      • 深拷贝引用类型数据，再做操作
    • 给定固定的输入，就一定会有固定的输出
      • new Date()
      • Math.random()
    • 副作用
      • 修改了传入的对象 （修改参数）
      • 在window上添加了属性
      • DOM API 修改页面
      • 发送http请求（ajax）
      • 调用 window.reload 刷新浏览器
      • console.log 往控制台打印数据

redux中间件

redux的中间件，其实就是对store.dispatch的一层封装，在“假”的dispatch中增加额外的代码，然后再内部调用真的dispatch来更新state

  • 简单实现

    redux-thunk

    • 功能： 异步代码

    • 使用：store.dispatch(action) , action可以是一个函数，该函数接收真正的dispatch作为参数

    • 配置 devtools 和 中间件 ： 见github/redux-devtools
